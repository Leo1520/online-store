⚠️ Notas Importantes:

  • La transacción asegura consistencia: si hay error, se hace ROLLBACK de todos los cambios
  • El número de venta se genera automáticamente (secuencial)
  • Cada producto del carrito genera un DETALLE de venta
  • El stock se descuenta por cada producto vendido
  • Si hay error en cualquier paso, toda la venta se cancela

📊 Relaciones:

• Un Cliente tiene UNA Cuenta de acceso
• Un Cliente genera múltiples NotasVenta
• Una NotaVenta contiene múltiples DetallesVenta
• Cada DetalleVenta referencia un Producto
• Cada Producto tiene stock por Sucursal

📝 Resumen Ejecutivo del Flujo

Flujo General del Proceso:

  1. Usuario accede a carrito: Revisa productos seleccionados
  2. Procede al pago: PagoControlador valida carrito y usuario logueado
  3. Registra venta: VentaModelo inicia transacción en BD
  4. Obtiene CI del cliente: Se busca en tabla Cliente
  5. Crea NotaVenta: Se genera número secuencial de venta
  6. Inserta detalles: Para cada producto, se registra cantidad y se descuenta stock
  7. Confirma transacción: COMMIT si todo es exitoso, ROLLBACK si hay error
  8. Redirige a éxito: Usuario ve confirmación con número de venta
  9. Admin consulta ventas: Puede ver listado y detalles de todas las ventas
