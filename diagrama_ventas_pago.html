📖 Descripción del Flujo:

Este es el proceso más crítico. Se utiliza una transacción para garantizar que o se registran todos los datos correctamente o no se registra nada. Si hay cualquier error (producto no existe, stock insuficiente, etc.), se deshacen todos los cambios.

🔒 Transacción Atómica (ACID)

  • Atomicidad: Todo se guarda o todo se cancela
  • Consistencia: La BD queda en estado válido
  • Aislamiento: Otros procesos no ven cambios incompletos
  • Durabilidad: Los datos se guardan de forma permanente

📋 Pasos Críticos:

1
Obtener CI del cliente Se busca el CI del cliente usando su usuario de sesión
2
Iniciar transacción BEGIN TRANSACTION - A partir de aquí, o todo entra o nada
3
Crear NotaVenta Se genera número secuencial para la venta y se registra
4
Procesar cada producto Para cada producto: validar, registrar detalle, descontar stock
5
Confirmar transacción COMMIT - Guardar todos los cambios de forma permanente

Después de registrar la venta, se limpia el carrito de la sesión para evitar duplicados. El usuario es redirigido a una página de confirmación donde se muestra el número de venta generado. Si algo falló, se muestra un mensaje de error.

⚠️ Posibles Errores Manejados:

  • Carrito vacío: No hay productos para vender
  • Usuario no identificado: No viene en sesión
  • Cliente no existe: No hay registro en tabla Cliente
  • Producto no existe: Se salta ese producto (no es error crítico)
  • Stock insuficiente: Se cancela toda la venta (ROLLBACK)
  • Sin productos válidos: Se cancela la venta si todos fallaron
